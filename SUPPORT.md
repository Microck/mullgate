# Support

## Where to ask for help

- GitHub Issues: bugs, regressions, packaging problems, and clear feature requests
- `docs/usage.md`: setup details, runtime behavior, and troubleshooting
- `SECURITY.md`: private vulnerability reporting

## What to include

When asking for help, include:

- Mullgate version
- operating system
- whether you installed from npm, a release asset, or a source checkout
- the command you ran
- the redacted output from `mullgate status` or `mullgate doctor` when relevant

Do not post real credentials or account numbers.

## Support expectations

This is a maintained open-source CLI, not a commercial support channel. High-signal repros and logs will get better answers faster than vague reports.
